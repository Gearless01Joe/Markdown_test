# 配置说明

NTRT 项目通过 `application/settings.py`、`base_mysql.py` 以及若干常量，提供灵活的部署选项。本页列举常用配置及修改建议。

---

## 1. 数据库配置（application/settings.py）

```python
DATABASES = {
    'medicine': {
        'HOST': '192.168.1.243',
        'PORT': '3306',
        'USER': 'medpeer',
        'PASSWORD': 'medpeer',
        'NAME': 'medicine',
        'CHARSET': 'utf8mb4',
    },
    'medicine_test': {
        'HOST': 'localhost',
        'PORT': '3306',
        'USER': 'root',
        'PASSWORD': '248655',
        'NAME': 'medicine_test',
        'CHARSET': 'utf8mb4',
    },
    ...
}
```

### 建议
- 生产/测试环境拆分为不同 key，例如 `medicine`、`medicine_test`
- 将敏感信息改为读取环境变量：`os.getenv("NTRT_DB_PASSWORD", "medpeer")`
- 如需新增数据源，只需追加配置即可被 `session_dict` 自动识别

---

## 2. Session 工厂（base_mysql.py）

模块在导入时遍历 `DATABASES`，为每个 key 创建 `SessionLocal` 并存入 `session_dict`。使用方式：

```python
from base_mysql import session_dict
session = session_dict['medicine']()
```

- `DEFAULT_DB_KEY` 默认为 `medicine`，可在运行前修改
- 如果担心连接过多，可用 `NullPool`/`QueuePool` 调整 SQLAlchemy pool 配置

---

## 3. DataCleaner 参数

| 参数 | 位置 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `BATCH_SIZE` | `data_cleaner.py` | 100 | 单次查询/写入的记录数 |
| `DEFAULT_DB_KEY` | `data_cleaner.py` | `medicine` | 选用的数据库别名 |
| `cleaning_plan` | `run()` | - | 描述需要清洗的字段、模型、处理函数 |

> 可将这些值抽象成配置文件或命令行参数，以便在不同场景切换。

---

## 4. JSON 规则与文档

根目录提供多份 `.json` 与 `.md` 文档（如《数据清洗流程说明》《文件调用关系说明》），可作为字段映射与验证的依据。若业务规则发生变化，请同步更新这些文档，以免处理逻辑与实际需求脱节。

---

## 5. 日志与调试

目前 `DataCleaner` 使用 `print` 输出，建议在以下场景切换至标准日志模块：

- 需要输出到文件或集中式日志系统
- 需要区分 INFO / WARNING / ERROR
- 需要追踪具体数据 ID

示例：

```python
import logging
logger = logging.getLogger(__name__)

logger.info("开始清洗 %s", display_name)
```

---

## 6. 环境变量示例

```bash
set NTRT_DB_HOST=192.168.1.243
set NTRT_DB_USER=medpeer
set NTRT_DB_PASSWORD=******
set NTRT_DB_NAME=medicine
```

在 `settings.py` 中读取：

```python
import os
DATABASES['medicine']['HOST'] = os.getenv('NTRT_DB_HOST', '192.168.1.243')
```

| 仅读取不写入 | `_write_updates` 中暂时跳过 `session.commit()` | 注意回滚 |

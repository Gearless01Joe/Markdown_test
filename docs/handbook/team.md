# 团队介绍

## 组织与职责

- **数据建设组**：负责数据采集、清洗、投递与可视化闭环。
- **项目 owner**：各重点项目（NTRT、RCSB PDB）均设产品/技术双负责人，协同确认需求与交付。
- **技术文档维护人**：每个模块至少一名 maintainer，负责 Page Review 与版本打标。

## 沟通渠道

| 场景 | 渠道 | 约定 |
| --- | --- | --- |
| 日常开发 | 企业微信群/飞书群 | 30 分钟内响应，告知进度或阻塞 |
| 需求/方案评审 | 周例会 + 飞书文档 | 会前 24h 共享材料，会议纪要归档到 `docs/` |
| 值班支持 | PagerDuty/电话 | 15 分钟内确认，严重级别按 S0/S1 流程 |
| 文档反馈 | GitHub Issues | 不同标签（`content`, `bug`, `request`）分类 |

## 新人成长路径

1. 通过 [快速开始](../guide/getting-started.md) 完成环境准备。
2. 参考 [第一小时任务](first-hour.md) 完成项目克隆、示例运行、首个 PR。
3. 阅读 [编码规范](../standards/index.md) 与对应项目手册。
4. 预约 Pair Session，由导师 walkthrough 核心代码与部署栈。

## 维护职责

- **内容审核**：模块 owner 每周巡检「最近更新」；超过 60 天未更新的页面要标注状态。
- **版本同步**：研发发布后 24h 内更新文档快照，必要时打 `vX.Y` 标签。
- **自动化脚本**：脚本更新后需同步 `scripts/` 说明，并在此处登记责任人。
